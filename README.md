# VigiDrive — AI Driver Monitoring System
### Vision & AI Core Module

> Real-time driver drowsiness and distraction detection using computer vision and ML.

---

## Project Structure

```
vigidrive/
│
├── main.py              ← Entry point — run this to start the system
├── train_model.py       ← CNN training script for eye classifier
├── evaluate.py          ← Offline evaluation on video files
├── requirements.txt     ← Python dependencies
│
├── core/
│   ├── detector.py      ← DriverMonitor: EAR, MAR, PERCLOS, head pose
│   └── alert.py         ← AlertSystem: multi-level alerts + audio + logging
│
├── utils/
│   └── display.py       ← Dashboard: HUD overlay renderer
│
├── models/              ← Trained model files go here
│   └── eye_classifier.h5   (generated by train_model.py)
│
├── data/                ← Dataset folder
│   └── eye_dataset/
│       ├── open/        ← Open eye images
│       └── closed/      ← Closed eye images
│
└── logs/                ← Auto-created session CSV logs
```

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Run the live monitor
```bash
python main.py
```

### 3. Run with options
```bash
python main.py --camera 0 --sensitivity high --save-log
```

| Flag | Description |
|---|---|
| `--camera N` | Camera index (0 = default webcam) |
| `--sensitivity` | `low` / `medium` / `high` |
| `--save-log` | Save events to `logs/` as CSV |
| `--no-display` | Headless mode (no window) |

## Features

### 👁 Eye Aspect Ratio (EAR)
Detects eye closure using the landmark-based EAR formula:

```
EAR = (|p2-p6| + |p3-p5|) / (2 * |p1-p4|)
```
- Normal range: **0.25 – 0.40**
- Below threshold (≈ 0.22) for N consecutive frames → drowsiness alert

### 👄 Mouth Aspect Ratio (MAR)
Detects yawning using similar landmark distances on the mouth region.
- MAR > 0.70 for sustained frames → yawn detected

### 📊 PERCLOS
Percentage of Eye Closure over a rolling 30-second window.
- Industry-standard drowsiness metric (NHTSA validated)
- PERCLOS > 25% → drowsiness warning

### 🔢 Blink Rate
Tracked over a 60-second rolling window.
- Normal: 15–20 blinks/min
- Anomaly (< 8 or > 30) → fatigue indicator

### 🗣 Head Pose Estimation
3D head orientation via `cv2.solvePnP` with 6 facial landmarks:
- **Pitch** (nodding down) → head drop alert
- **Yaw** (looking sideways) → distraction alert

### 🧠 Drowsiness Level (0–3)
Multi-factor scoring combining EAR, PERCLOS, yawning, head pose, blink rate:
```
Level 0 → ALERT
Level 1 → MILD FATIGUE
Level 2 → DROWSY        → audio warning
Level 3 → DANGER        → urgent alert + pulsing red overlay
```

## Training Your Own Model

### Dataset options
- **CEW** – Closed Eyes in the Wild
- **MRL Eye Dataset** – 84,898 annotated eye images  
- **NTHU Drowsy Driver Dataset**

### Prepare dataset
```
data/eye_dataset/
├── open/     ← .jpg/.png open eye images
└── closed/   ← .jpg/.png closed eye images
```

### Train
```bash
python train_model.py --data_dir data/eye_dataset --epochs 30
```
Model saved to `models/eye_classifier.h5`

---

## Evaluate on Video
```bash
python evaluate.py --video test_videos/driver1.mp4 --sensitivity medium
```
Outputs per-frame JSON with all metrics to `results/eval_results.json`

## Tech Stack

| Component | Technology |
|---|---|
| Face detection & landmarks | MediaPipe FaceMesh (468 pts) |
| Eye/Mouth metric | EAR / MAR (scipy spatial) |
| Head pose | OpenCV solvePnP |
| CNN classifier | TensorFlow / Keras |
| Video processing | OpenCV |
| Audio alerts | Pygame |
| Fallback detector | dlib (optional) |

---

## Alert Levels

| Level | Color | Trigger |
|---|---|---|
| INFO | Green | Mild fatigue / yawn |
| WARNING | Orange | Drowsy / distracted |
| CRITICAL | Red | Severe drowsiness |

# Driver-Drowsiness-Behavior-Monitoring
Real-time AI-based driver drowsiness and behavior monitoring system using computer vision, sensor fusion, and multi-level alerts.
AI Driver Monitoring System

A real-time driver safety system that detects drowsiness, 
distraction, and unsafe driving behaviour using computer 
vision and vehicle sensor fusion.

Built as a two-module system:
- Vision & AI Core — MediaPipe FaceMesh, EAR, MAR, 
  PERCLOS, head pose estimation
- Integration & Alerts — Sensor fusion, risk scoring, 
  multi-level alert engine, live dashboard, CSV logging

Tech Stack:
Python · OpenCV · MediaPipe · TensorFlow · Scikit-learn
NumPy · SciPy · Pygame

Features:
- Real-time face mesh with 468 landmarks
- Drowsiness scoring (Level 0–3)
- Vehicle sensor data integration (speed, braking, 
   acceleration)
- Unified risk score (0–1)
- 3-level alert system (INFO / WARNING / CRITICAL)
- Live web dashboard
- Per-session CSV logging & evaluation report
